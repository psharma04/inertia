# Protocol Compatibility Fixtures

This directory contains deterministic binary test fixtures generated by the
Python Reticulum and LXMF reference implementations.

Swift tests in `ReticulumCryptoTests`, `ReticulumPacketsTests`, and `LXMFTests`
load these fixtures to verify byte-for-byte binary compatibility between the
Swift implementation and the Python reference.

---

## Directory Structure

```
Tests/ProtocolFixtures/
├── identities/        Identity key derivation and hashing
├── destinations/      Destination hash computation
├── packets/           Packet wire-format encoding
├── announces/         Announce packet encoding and signature
└── lxmf/             LXMF message envelope encoding
```

---

## Fixture Format

Each fixture is a JSON file with three top-level keys:

```json
{
  "description":  "Human-readable explanation of what is being tested",
  "rns_version":  "1.1.3",
  "inputs":  { ... },
  "expected": { ... }
}
```

All binary data is hex-encoded (`"deadbeef"`).
Integer values are plain JSON numbers.
String values are UTF-8.

---

## Regenerating Fixtures

Run whenever the protocol implementation changes or a new fixture type is
needed.

### Prerequisites

```
pip install rns lxmf
```

### Run

```
python Tests/Scripts/generate_fixtures.py
```

Fixtures are written to `Tests/ProtocolFixtures/`.  **Commit the updated fixtures
alongside any Swift implementation changes** so the two stay in sync.

---

## Private Key Seeds

Deterministic test-only private keys are used during fixture generation and in
test helper code:

| Label        | Private key bytes |
|---|---|
| `identity_a` | `0x00 … 0x3F` (sequential) |
| `identity_b` | `0x40 … 0x7F` (sequential) |

> **Warning:** These keys are public knowledge and exist solely to make tests
> reproducible.  They are **not** stored in fixture JSON.

## Using Fixtures in Swift Tests

Load a fixture file and assert against the `expected` fields:

```swift
import Testing
import Foundation
@testable import ReticulumCrypto

@Test("identity_a hash matches reference")
func identityAHashMatchesReference() throws {
    let url = Bundle.module.url(
        forResource: "identity_a",
        withExtension: "json",
        subdirectory: "Tests/ProtocolFixtures/identities"
    )!
    let fixture = try JSONDecoder().decode(IdentityFixture.self, from: Data(contentsOf: url))
    let privateKey = DeterministicIdentityKeyMaterial.identityAPrivateKey
    let identity = try Identity(privateKey: privateKey)
    #expect(identity.hash == Data(hexString: fixture.expected.identityHashHex)!)
}
```

## Fixture Inventory

| File | Description |
|---|---|
| `identities/identity_a.json` | Public key and hash vector for deterministic identity A |
| `identities/identity_b.json` | Public key and hash vector for deterministic identity B |
| `destinations/destination_single_identity_a.json` | SINGLE destination hash for `inertia.test` |
| `destinations/destination_single_identity_b.json` | SINGLE destination hash for `inertia.test` (identity B) |
| `destinations/destination_plain.json` | PLAIN destination hash for `inertia.broadcast` |
| `packets/data_packet_plain.json` | Wire-format DATA packet to a PLAIN destination |
| `announces/announce_basic.json` | Signed ANNOUNCE packet with app_data |
| `lxmf/message_basic.json` | LXMF message envelope with fixed timestamp |
